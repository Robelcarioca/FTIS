# FTIS - Flight Turbulence Intelligence System

FTIS is an AI-powered aviation turbulence intelligence platform. It gathers live aircraft and atmospheric data, engineers turbulence-specific aviation features, trains supervised ML models, exposes real-time predictions through FastAPI, and visualizes route-level turbulence risk in a Streamlit operations dashboard.

The project is designed as a production-style MVP: research-grade enough to show ML rigor, modular enough to scale, and polished enough for a startup-quality aerospace AI portfolio.

## Core Capabilities

- Live aircraft ingestion from OpenSky state vectors.
- Atmospheric enrichment from Open-Meteo weather data.
- Flight Turbulence Index (FTI), a proprietary 0-100 turbulence risk score.
- Aviation feature engineering for wind shear, temperature gradient, pressure variation, altitude bands, atmospheric instability, and turbulence intensity.
- Baseline, Random Forest, and XGBoost training pipeline with model comparison.
- Evaluation reports with classification metrics, confusion matrix, and ROC curves when available.
- Async live weather integration for Open-Meteo, NOAA/NWS, and AviationWeather.gov with cache and provider failover.
- Route simulation engine with waypoint interpolation, altitude profile generation, weather sampling, turbulence corridor detection, GeoJSON, and CSV export.
- Explainability and monitoring utilities for SHAP summaries, feature importance, calibration, confidence, and drift reporting.
- FastAPI service for prediction, batch scoring, route analysis, live weather, model metrics, and system diagnostics.
- Dark, aviation-grade Streamlit operations center with route playback, weather overlays, heat zones, confidence gauges, alerts, and model analytics.
- Docker, Docker Compose, environment config, CI/CD skeleton, registry, linting, and tests.

## Architecture

```text
OpenSky API -> data/raw/flights.csv
Open-Meteo API -> data/weather/weather.csv
Flight/weather merge -> data/processed/training_data.csv
Feature engineering -> data/processed/features.csv
Model training -> models/ftis_model.pkl
Evaluation -> docs/model_results/
FastAPI -> /predict, /predict/batch, /route/analyze, /weather/live, /model/metrics, /system/status
Streamlit -> aviation operations center
```

## API Usage

Start the API:

```bash
uvicorn api.main:app --reload
```

Endpoints:

- `GET /`
- `GET /health`
- `POST /predict`
- `POST /predict/batch`
- `POST /route/analyze`
- `POST /weather/live`
- `GET /model/metrics`
- `GET /system/status`
- Versioned mirrors under `/api/v1/*`

Example request:

```bash
curl -X POST http://localhost:8000/predict \
  -H "Content-Type: application/json" \
  -d '{
    "latitude": 39.8729,
    "longitude": -104.6737,
    "altitude": 10800,
    "windspeed": 42,
    "pressure": 1002,
    "temperature": -42
  }'
```

Example response:

```json
{
  "risk": "High",
  "confidence": 0.91,
  "FTI": 82.0,
  "recommendation": "Avoid route section or request tactical reroute.",
  "probabilities": {
    "Low": 0.03,
    "Moderate": 0.06,
    "High": 0.91
  }
}
```

Interactive API docs are available at `http://localhost:8000/docs`.

## Research Motivation

Aircraft turbulence remains a safety, comfort, routing, and fuel-planning challenge. FTIS frames turbulence prediction as a fusion problem across aircraft kinematics and atmospheric state. The MVP uses interpretable engineered indicators such as wind shear, pressure variation, temperature gradient, and altitude regime, then compares traditional machine learning models before serving the best model for operational decision support.

The initial FTI metric is intentionally transparent: it produces a stable 0-100 turbulence index that can support research iteration, cockpit/dispatch visualization, and future calibration against pilot reports, EDR, AMDAR, or radar-derived turbulence observations.

## Future Roadmap

- Add live turbulence ground truth from PIREPs, EDR, AMDAR, or NOAA products.
- Add geospatial tiling and vector-tile serving for large-scale turbulence zones.
- Introduce temporal models such as gradient boosted lag features, LSTM, or transformer encoders.
- Add model monitoring for class drift, feature drift, and calibration error.
- Move ingestion into scheduled orchestration with Airflow, Prefect, or managed cloud workflows.
- Add route optimization that proposes altitude and lateral reroute alternatives.
- Add authentication, audit logs, and role-based access for dispatch users.
